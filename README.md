# نورتیکا (Noortika) 🌙✨
**نوری در مسیر تاریکی تو!**

یک وب‌اپلیکیشن کامل برنامه‌ریزی تحصیلی هوشمند برای دانش‌آموزان متوسطه دوم (پایه‌های دهم، یازدهم و دوازدهم)
با بک‌اند FastAPI (پایتون) و فرانت‌اند سرور-رندر (Jinja2 + JS خالص).

## امکانات
- فرم ثبت‌نام (نام، تلفن، گروه تحصیلی، رشته هدف، دانشگاه هدف، ساعت مطالعه)
- موتور هوشمند ساخت برنامه درسی روزانه بر اساس پایه/رشته/هدف دانشگاهی (قابل ارتقا به هوش مصنوعی واقعی)
- امکان افزودن درس دلخواه توسط کاربر
- موتور تولید آزمون تستی شبیه‌ساز کنکور (۵ تا ۳۰ سوال)
- رتبه‌بندی بر اساس ساعت مطالعه + نتایج آزمون، با جایزه نمادین برای سه رتبه برتر
- بخش پشتیبانی (تیکت) و سوالات پرتکرار
- پلن‌های اشتراک (هفتگی رایگان / ماهانه / شش‌ماهه / یک‌ساله) با **پرداخت نمادین** (بدون درگاه واقعی)
- تم روشن و تاریک با رنگ‌بندی شاد و پویا

## درباره موتور هوش مصنوعی
به‌صورت پیش‌فرض، سیستم از یک **موتور هوشمند قانون‌محور داخلی** (Python) برای ساخت برنامه و آزمون استفاده می‌کند
که کاملاً به سرفصل‌های متوسطه دوم ایران آشناست و بدون نیاز به هیچ سرویس خارجی کار می‌کند.

اگر بخواهید از هوش مصنوعی مولد واقعی (Claude) هم برای غنی‌سازی توصیه‌ها و طراحی سوالات استفاده شود،
کافی است متغیر محیطی `ANTHROPIC_API_KEY` را در Render (یا فایل `.env`) تنظیم کنید؛ در غیر این‌صورت
همه‌چیز با موتور داخلی و بدون خطا کار می‌کند.

## اجرای محلی
```bash
pip install -r requirements.txt
uvicorn main:app --reload
```
سپس به آدرس `http://localhost:8000` بروید.

## اجرا با Docker
```bash
docker build -t noortika .
docker run -p 8000:8000 noortika
```

## دیپلوی روی Render
1. مخزن (ریپازیتوری) را روی گیت‌هاب/گیت‌لب آپلود کنید.
2. در Render یک **Web Service** جدید بسازید و گزینه **Environment: Docker** را انتخاب کنید (فایل `Dockerfile` به‌صورت خودکار شناسایی می‌شود؛ فایل `render.yaml` هم برای Blueprint آماده است).
3. نیازی به تنظیم متغیر PORT نیست؛ Render خودش آن را تزریق می‌کند و پروژه از آن استفاده می‌کند.
4. (اختیاری) متغیر `ANTHROPIC_API_KEY` را در بخش Environment اضافه کنید تا موتور هوش مصنوعی واقعی فعال شود.
5. Deploy کنید — بدون نیاز به هیچ تنظیم اضافه.

## ساختار پروژه
```
noortika/
├── main.py                 # ورودی اصلی FastAPI
├── requirements.txt
├── Dockerfile
├── render.yaml
├── app/
│   ├── database.py          # اتصال SQLite
│   ├── models.py             # مدل‌های دیتابیس
│   ├── schemas.py            # مدل‌های ورودی/خروجی API
│   ├── curriculum.py         # سرفصل دروس دهم تا دوازدهم
│   ├── ai_planner.py          # موتور هوشمند ساخت برنامه درسی
│   ├── ai_quiz.py             # موتور تولید آزمون تستی
│   └── routers/               # روت‌های API (کاربران، برنامه، آزمون، رتبه‌بندی، پشتیبانی، اشتراک)
├── templates/                 # صفحات HTML (Jinja2)
└── static/                    # CSS و JS
```

## نکات مهم
- پایگاه‌داده SQLite است و در پوشه `data/` ذخیره می‌شود؛ در پلن رایگان Render دیسک پایدار نیست و با هر ری‌استارت داده‌ها پاک می‌شود (برای دموی نمادین مناسب است).
- پرداخت اشتراک کاملاً **نمادین** است؛ هیچ درگاه بانکی واقعی متصل نشده است.
